# Planeación

1. Sin planes beunos, administrar incluso equipos pequeños es casi iposible
2. La planeación es una habilidad que puedes aprender y mejorar con la práctica
3. La buena planeación lleva a mejor software

Un buen plan sirve como una herramienta de comunicación y negociación para generar compromisos en tiempo y forma.

## Plan de proyecto

Un plan de proyecto define el trabajo y cómo se realiza. Provee definiciones de las tareas principales, estimados de tiempo y recursos, así como un marco de referncia para la revisión y control. Su documentación adecuada permite un punto de partida para la estimación real de rendimiento.

En una org madura, un plan se usa para:

1. Establecer una base para los acuerdos de costos y tiempos de un trabajo
2. Una organización estructurada para realizar dichos trabajos
3. Un punto de referencia para obtener los recursos necesarios
4. Un registro de lo que se prometió inicialmente

## Funciones de un plan

- Acomodo en tamaños de la(s) actividad(es)
- Estructuración de la actividad. ¿Cómo se realiza y en qué orden?
- Estatus. Cómo sabes dónde vas? ¿Terminarás en tiempo y costos?
- Evaluación. ¿Qué tan bueno es tu plan? ¿Cometiste errores? ¿Cómo los corriges?

## Planes con stakeholders

Tus stakeholders probablemente quieren:

1. ¿Qué se prometió? ¿Cuándo, dónde y a qué costo?
2. ¿Qué tan bueno es el producto? ¿Es lo deseado? ¿tiene calidad? ¿deleita al cliente?
3. ¿Existe una manera de mostrar progreso? ¿Se pueden visualizar los costos de calidad, tiempo y/o costo de manera temprana?
4. ¿Permite evaluar qué tan bien se realizó todo? ¿Se pueden separar los problemas de mala planeación de los de mala administración? ¿El impacto de camnio de alcance es visible?

## Partes necesarias de un buen plan

1. El plan se basa en piezas definidas de trabajo. Sin la definición de estas piezas de trabajo, no hay manera de haacer un buen plan ni un buen producto.
2. El trabajo implica múltiples pasos claramente definidos y medibles. Esto permite la medición de progreso.
3. Revisa el plan con tu usuario antes de trabajar. Esta acción es esencial para toda tarea, excepto las más triviales.
4. Deberás realizar enunciamientos periódicos del progreso a tus cientes. Esto genera confianza y hace sentir al cliente involucrado.
